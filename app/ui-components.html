App Bar

The app bar component is used as header for apps.

Just like most other components it scales to the column it's in.

Properties includes an array for nav items and an optional one for one or more action buttons. 

$(ui.component.appbar.render({
  'navItems': [{
    'url': '#',
    'item': 'Past events',
    'class': 'nav--active divider'
  }, {
    'url': '#',
    'item': 'Upcoming events'
  }],
  'button': [{
    'class': 'solid-button--strict pull-right',
    'content': 'Add new event'
  }]
}));

Messages

There are two types of messages. Alerts & Dialogs

  • Alerts are used to provide a user with feedback
  • Dialogs are used when user input is needed for an action

Alerts

You can either use the solid or the naked alert.
The 'class' property defines the alert style. Possible options are:

  • naked-alert--happy
  • naked-alert--strict
  • naked-alert--angry

The solid alert stands out more and is to be used for important messages only.
If the alert message is not of high importance the use of the naked alert is prefered.

Dialogs

An dialog is presented to a user when immediate user input is required.

There are two types of dialogs:

  • Strict dialog (automaticly selected)
  • Angry dialog
    ( 'angry': 'true')
The angry dialog is only to be used for high impact decisions, like deleting an account. 


$(ui.component.dialog.render({
    angry: 'true',
    title: 'Deleting this is permanent.',
    message: 'Are you sure you want to delete your account?',
    button: [{
        content: 'Delete account'
    }, {
        content: 'Cancel'
    }]
}))
