Layout

Grid

Right now we use the Sass version of Bootstrap's grid. Note we only use Bootstrap's grid. Other bootstrap classes don't work.

Most important rules:

  • Do not nest containers
  • Rows must be placed within a .container (fixed-width) or .container-fluid (full-width) for proper alignment and padding
  • Use rows to create horizontal groups of columns
  • Content should be placed within columns, and only columns may be immediate children of rows

More information on:

getbootstrap.com/css/#grid getbootstrap.com/examples/grid 


<div class="container"> 
    <div class="row"> 
        <div class="col-md-4"> 
            <!--content-->    
        </div> <!--/col-md-4-->
        <div class="col-md-8">

</div> <!--/container-->

Platform Layout

Mr.Watson uses a 3 column grid with a full-width header on top
The most right column is optional. If not used the middle column becomes (col-md-9) and takes up the extra space.

The left column (col-md-3) is for navigation and disappears on small devices or screen-widths.

The middle column (col-md-6) is for main content.

The (optional) right column (col-md-3) is for widgets and disappears on small devices or screen-widths.

App Layout

Apps can use the 9 columns on the right in the same fashion the platform does. They can either take up the entire 9 columns. I.e. for placing the App Bar Component which houses the app's filter views and action buttons.

The optional right column is used for in-app navigation.
